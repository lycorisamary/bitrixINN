Краткий анализ проекта bitrixINN

1. Проект представляет собой локальное PHP-приложение для Bitrix24.
2. Основные файлы:
   - config.php: базовая конфигурация, пути к auth-файлам, SSL и режим admin auth.
   - install.php: принимает install-данные Bitrix24, сохраняет токены и регистрирует placement.bind.
   - handler.php: основной обработчик интерфейса и логики поиска компаний по ИНН.
3. В handler.php есть расширенное логирование в runtime_debug_handler.log.
4. По текущему логу видно, что приложение успешно извлекает пользователя из direct keys:
   - member_id = 76
5. Также видно, что часть запросов приходит с PLACEMENT_OPTIONS, а часть без них.
6. Логика получения пользователя в handler.php сделана с несколькими fallback:
   - из PLACEMENT_OPTIONS
   - из прямых request-параметров
   - через user.current
7. Для поиска компаний используются REST-методы Bitrix24 CRM.
8. Для назначения наблюдателя используется обновление CRM-компании через observers.

Замечания:
- README отображается в некорректной кодировке в терминале, но сам проект выглядит рабочим.
- В config.php verify_ssl сейчас выключен (false), это удобно для локальной разработки, но рискованно для production.
